# Novel Chinese TTS Design

## Goal

Make Traditional Chinese novel narration match the natural ElevenLabs preview more closely without changing English pronunciation behavior.

## Behavior

- Chinese novel narration uses voice ID `r6qgCCGI7RWKXCagm158`.
- Chinese requests use `eleven_multilingual_v2`.
- Chinese audio always plays at `1.0x`.
- English requests keep the existing voice, model, and user-selected playback speed.
- Only Chinese speech explicitly marked as novel API TTS uses this path. Other Chinese speech continues using browser speech.

## Data Flow

1. `NovelM` marks Chinese novel utterances with `apiTts: true`.
2. The browser patch sends `lang: zh-TW` and a Chinese playback speed of `1.0`.
3. The Netlify function selects the dedicated Chinese voice and Chinese model.
4. The cache key includes voice ID, model ID, and voice settings.
5. Supabase Storage is checked before ElevenLabs is called.
6. A cache miss is generated by ElevenLabs and uploaded to the `tts-cache` bucket.

Existing cache entries are not overwritten. Changing the voice and model creates new cache paths naturally.

## Configuration

- `ELEVENLABS_ZH_VOICE_ID=r6qgCCGI7RWKXCagm158`
- `ELEVENLABS_ZH_MODEL_ID=eleven_multilingual_v2`

The existing `ELEVENLABS_MODEL_ID` remains the English/default model.

## Verification

- Unit test that Chinese requests use the dedicated voice and Chinese model.
- Unit test that Chinese novel audio plays at `1.0x` even when the English preference is `0.9x`.
- Confirm a production request returns ElevenLabs audio on the first uncached request.
- Confirm the same request subsequently returns `X-TTS-Source: supabase-cache`.
- Confirm English TTS behavior is unchanged.
